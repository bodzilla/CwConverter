ChaoticWars Converter

How it works

Overview

This page is a live value converter with two aligned columns: Base Values on the left and Converted Values on the right.

Each row is intentionally paired like this:

  • Reset Values ↔ Money
  • $ / Crystal ↔ Crystals
  • $ / Step ↔ Steps
  • $ / Drug ↔ Drugs
  • $ / Potion ↔ Potions
  • $ / Donator Pack ↔ Donator Packs
  • $ / Item ↔ Items

Base values

The left column defines the dollar value for each commodity.

For example, CrystalBase = 35 means 1 Crystal = $35, and ItemBase = 100 means 1 Item = $100.

The Item row is slightly special because the word Item is editable, but the row still stays aligned with the converted Items field.

Converted values

The right column is where the user enters a value to convert.

Whichever converted field the user types into becomes the active source value, and the rest are recalculated from it.

Typing into Money calculates the equivalent number of Crystals, Steps, Drugs, Potions, Donator Packs, and Items. Typing into any other commodity converts back through Money and then out to the rest.

Calculation logic

The calculations use a shared Money value behind the scenes.

Example: if the user enters 10 Crystals, the app first calculates Money = Crystals × CrystalBase, then uses that Money value to calculate the other commodities.

This keeps every conversion consistent with the same pricing system.

Reset behaviour

The Reset Values button restores the defaults for all base values and the Item label, then clears the converted values back to zero.

  • CrystalBase = 35
  • StepBase = 220,000
  • DrugBase = 300,000
  • PotionBase = 180,000
  • DonatorPackBase = 7,000,000,000
  • ItemBase = 100
  • ItemTextValue = Item

Number formatting

All numeric inputs keep comma separators while typing, so 1000 becomes 1,000 and 1000000 becomes 1,000,000.

The page treats numeric fields as text inputs, strips out commas and other non-numeric characters, performs the calculation, then writes the values back in formatted form.

The script also tries to preserve the cursor position while reformatting.
